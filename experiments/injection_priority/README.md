# Injection Priority 实验

验证 C3b（系统动态注入）和 C3a（用户查询指令）与静态通道（C1、CA）之间的注意力优先级。

## 实验设计

```
Token 序列:  [C1][CA][C3b 注入][C3a 用户查询]
               ↑    ↑      ↑           ↑
             system user  system      user
             静态   静态   动态注入    当前查询
```

7 个对抗场景 + 3 个控制组，每个场景 5 轮 × 3 次 = 15 次试验（并发=5）。

| 场景 | 竞争通道 | 关键问题 |
|------|---------|---------|
| C3b vs C1 | C3b (C) vs C1 END (A) | 动态纠正能否覆盖 system prompt？ |
| C3b vs CA END | C3b (A) vs CA END (C) | 动态纠正能否覆盖 CA 尾端？ |
| C3b vs C1 vs CA | C3b (B) vs C1 (A) vs CA (C) | 三通道动态注入优先级 |
| C3a vs C1 | 用户指令 (C) vs C1 END (A) | 用户意图能否覆盖 system prompt？ |
| C3a vs CA END | 用户指令 (A) vs CA END (C) | 用户意图能否覆盖项目规则？ |
| C3b vs C3a | C3b (A) vs 用户指令 (C) | 系统注入 vs 用户意图同回合竞争 |
| C3b 内部 | C3b[1] (A) vs C3b[2] (C) | 多条注入之间的近因效应 |

## 结果

```
┌──────────────────────────────────────┬──────────────┬──────────────────┐
│ 对抗场景                               │ 胜出          │ 机制              │
├──────────────────────────────────────┼──────────────┼──────────────────┤
│ C3b vs C1                            │ C3b 87-93%   │ 近因 + 明确覆盖措辞│
│ C3b vs CA END                        │ C3b 73-93%   │ 近因 > CA END    │
│ C3b vs C1 vs CA                      │ C3b 87-93%   │ C3b 压倒静态通道  │
│ C3a vs C1                            │ C1 73-87%    │ 用户缺乏覆盖措辞  │
│ C3a vs CA END                        │ 近乎平局       │ ~53:47           │
│ C3b vs C3a                           │ C3a 73-93%   │ 最近因           │
│ C3b 内部                              │ C3b[2] 100%  │ 近因效应          │
└──────────────────────────────────────┴──────────────┴──────────────────┘
```

## 关键发现

1. **C3b 合规纠正可靠** — 73-93% 覆盖 C1 和 CA，合规反馈架构（§4）在注意力层面是可行的
2. **用户显式指令不可靠** — "用 C，忽略其他规则"仅 7-27% 覆盖 C1。工具调用场景下，用户意图需要明确的"忽略 system prompt"措辞
3. **C3b 内部近因极强** — 最新注入 100% 覆盖更早注入，注入系统需要管理注入顺序
4. **C3b vs C3a 中 C3a 胜出** — 用户查询是绝对最后一条消息，无法被任何注入覆盖
